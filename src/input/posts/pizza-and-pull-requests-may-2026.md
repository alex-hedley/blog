#### Simon Emms: Temporal

**Title**: Failure Happens. Your Code Shouldn't Care.

**Description**: As software engineers, we know our code will be retried, restarted and run more than once. Temporal makes this manageable by letting us write reliable workflows that recover automatically from crashes and restarts without defensive plumbing. I’ll explain how it works and prove it with a live demo that treats failure as just another code path.
